## 1. 通用 SOP（每个 slice 必须遵守）

承接 bridge 删除经验，每个 slice **5 步执行**：

```
1. BASELINE
   git status --short          # 必须干净
   git log --oneline -5         # 确认起点

2. CHANGE
   按 slice 目标改文件（>10 文件改动 → 二次拆分）
   git diff --stat 自审

3. VERIFY（关键，不可省）
   - 必跑：python3 scripts/smoke_check.py（5 基础 smoke）
   - 必跑：./run-mossen.sh --help（启动链不崩）
   - TUI 改动必跑：expect(1) 模拟 TTY 真渲染
   - 状态/类型改动必跑：grep -rn "<删的字段>" --include="*.ts" --include="*.tsx" 全仓
   - 大改动必跑：python3 scripts/smoke_check.py（10 扩展 smoke + 真实 LLM agentic_tool_loop_runtime_audit）

4. COMMIT
   git add <显式路径列表>     # 禁用 git add . / git add -A（保护 commands/insights.ts WIP）
   git commit -m "<type>(<scope>): <slice 名> — <要点>

   Slice N/M of <task name>"

5. REVERT（失败时）
   git checkout -- <files>
   或 git reset --hard <prev-commit>
```

### 安全规则（不可破）

- ❌ **永不 `git push`**（除非用户明说）— `main` 领先 `origin` 是正常状态
- ❌ **永不 `git add .` / `git add -A`** 在仓库根目录（保护 `commands/insights.ts` WIP）
- ❌ **永不动 `commands/insights.ts`**（用户 WIP，永久保护）
- ❌ **不 mock 数据库 / 不跳过 hooks**（`--no-verify`）— bridge 删除被咬过
- ✅ 真删字段前 **必须 grep 全仓所有 read 点**（参考 memory `feedback_state_field_deletion`）
- ✅ TUI 启动验证 **必须用 expect(1) 模拟 PTY**，`mossen </dev/null` 走的是 print mode 不是 TUI

### 失败时的回退路径

| 失败级别 | 回退命令 |
|---|---|
| 单 slice 失败 | `git checkout -- <files>` 重做该 slice |
| 单任务失败 | `git reset --hard <task 起点 commit>` |
| 整轮失败 | `git reset --hard pre-upgrade-baseline-20260424` |
| 想看 bridge 老代码 | `git show pre-upgrade-baseline-20260424:bridge/<file>` |
